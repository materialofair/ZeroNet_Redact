# 双相册架构设计方案

**版本**: V1.0  
**日期**: 2025-01-19  
**目标**: 实现"导入Tab显示原始文件，相册Tab显示脱敏文件"的清晰架构

---

## 📋 1. 需求分析

### 1.1 核心需求
- **导入Tab**: 显示用户导入的所有原始图片和PDF（加密存储）
- **相册Tab**: 显示用户打码/脱敏后的文件（明文存储）
- **设置Tab**: 保持现有功能不变

### 1.2 非功能需求
- 原始文件必须加密存储（AES-256）
- 脱敏文件可以明文存储（便于分享）
- 数据模型清晰，职责分明
- 支持一个原始文件对应多个脱敏版本

### 2.2 Tab职责划分

#### 📥 导入Tab (ImportView)
**职责**: 
- 文件导入入口（相册选择器、文档选择器）
- 显示所有原始文件（网格视图）
- 点击文件进入脱敏编辑器

**功能**:
- ✅ 从系统相册导入图片
- ✅ 从文件系统导入PDF
- ✅ 自动加密保存到`OriginalFile`
- ✅ 显示导入历史（原始文件列表）
- ✅ 点击原始文件重新编辑/查看

#### 🖼️ 相册Tab (AlbumView)
**职责**:
- 展示所有脱敏完成的文件
- 文件分享、导出功能
- 查看脱敏文件详情

**功能**:
- ✅ 显示所有`RedactedFile`（网格视图）
- ✅ 文件类型过滤（全部/图片/PDF）
- ✅ 分享到系统（微信、小红书等）
- ✅ 查看文件详情（大小、导出时间）
- ❌ 不显示原始文件

#### ⚙️ 设置Tab (SettingsView)
**职责**: 保持不变
- 存储空间管理
- 隐私设置
- 关于信息

## 📊 3. 数据流设计

### 3.1 核心数据流

```
用户导入图片/PDF
    ↓
[ImportViewModel]
    ├─ 1. 读取文件数据
    ├─ 2. 使用CryptoEngine加密
    ├─ 3. 保存到本地加密存储
    └─ 4. 创建OriginalFile实体
    ↓
显示在导入Tab列表中
    ↓
用户点击文件
    ↓
[SimpleBrushEditor]
    ├─ 1. 加载OriginalFile
    ├─ 2. 解密显示原图
    ├─ 3. 用户手动/智能脱敏
    └─ 4. 保存脱敏结果
    ↓
创建RedactedFile实体（关联原文件）
    ↓
显示在相册Tab列表中
    ↓
用户分享脱敏文件
    ↓
系统分享面板
```

### 3.2 数据关系

```
OriginalFile (1) ────┐
                     ├─── RedactedFile (N)
                     │
                     ├─── RedactedFile (历史版本1)
                     │
                     └─── RedactedFile (历史版本2)
```

**关系说明**:
- 一个原始文件可以有多个脱敏版本
- 每次重新编辑同一原始文件，会创建新的`RedactedFile`
- 保留历史版本便于对比和回退

## 🗄️ 4. 技术选型

### 4.1 数据存储

| 层级 | 技术方案 | 用途 |
|------|---------|------|
| **数据模型** | Core Data | 持久化存储文件元数据 |
| **文件加密** | CryptoKit (AES-256-GCM) | 原始文件加密 |
| **密钥存储** | Keychain | 安全存储加密密钥 |
| **缩略图缓存** | ImageCache (内存LRU) | 网格视图性能优化 |

### 4.2 UI框架

| 组件 | 技术方案 |
|------|---------|
| **导航结构** | TabView (SwiftUI) |
| **文件列表** | LazyVGrid (SwiftUI) |
| **编辑器** | UIKit包装的DrawingCanvas |
| **文件选择** | PhotosPicker + UIDocumentPickerViewController |

### 4.3 架构模式

- **MVVM**: ViewModel管理业务逻辑
- **Repository模式**: StorageManager统一存储访问
- **加密层**: CryptoEngine独立加密模块

## 📦 6. 部署方案

### 6.1 存储目录结构

```
App Document Directory/
├── Originals/          # 原始文件（加密）
│   ├── {uuid}.enc      # 加密的图片/PDF
│   └── {uuid}.thumb    # 缩略图
├── Redacted/           # 脱敏文件（明文）
│   ├── {uuid}.png      # 脱敏后的图片
│   └── {uuid}.pdf      # 脱敏后的PDF
└── ZeroNetRedact.sqlite # Core Data数据库
```

### 6.2 全新数据模型

**说明**: 由于App尚未上线，直接使用新架构，无需考虑数据兼容性

**优势**:
- ✅ 简化实施流程
- ✅ 避免迁移代码复杂度
- ✅ 数据结构更清晰
- ✅ 减少技术债务

### 6.3 性能优化

**缓存策略**:
- ✅ 缩略图内存缓存（ImageCache, LRU, 最大100MB）
- ✅ 解密数据按需加载（不预加载全部）
- ✅ 网格视图懒加载（LazyVGrid）

**加密性能**:
- ✅ 后台线程加密（不阻塞UI）
- ✅ 使用硬件加速的AES-GCM
- ✅ 大文件分块处理（避免内存峰值）
