## 1. 产品名称

**中文名**：零网络脱敏  
**英文名**：ZeroNet Redact

- **ZeroNet**：继承"零网络"理念，强调完全离线、本地处理
- **Redact**：图像/信息脱敏的标准专业术语

---

## 2. 产品定位（One-liner）

**ZeroNet Redact 是一款完全离线的隐私文件脱敏工具，通过智能识别+手动编辑对图片和PDF进行脱敏处理，并提供双相册架构安全存储原文件与脱敏文件。**

### 🎯 核心定位关键词

- ✅ **隐私文件脱敏器**（核心功能）
- ✅ **图片脱敏（V1.0）+ PDF脱敏（V1.5+）**
- ✅ **本地文件库**（辅助功能，不强调）
- ✅ **原文件加密存储 + 脱敏文件明文存储**（双相册架构）
- ✅ **可扩展架构**：支持未来扩展视频、Word等格式
- ❌ 不是隐私空间（不做访客模式、伪装密码等）
- ❌ 不做视频脱敏（V1.0/V1.5，V2.0可选）
- ❌ 不强调文件管理（以工具为主，相册为辅）

---

## 3. 产品使命（Mission）

**在一个所有隐私工具都依赖云端的时代，ZeroNet Redact 用本地处理与加密技术，让图片脱敏真正做到"不会上传、不可能泄露"，同时通过双相册架构保护原图安全。**

## 4. 核心架构（双相册设计）🆕

### 📂 双相册架构

```
┌─────────────────────────────────────────┐
│         ZeroNet Redact 架构              │
├─────────────────────────────────────────┤
│                                         │
│  📸 导入原图（系统相册/拖拽）            │
│           ↓                             │
│  🔒 自动加密存入「原相册」               │
│     （AES-256加密，密钥存Keychain）      │
│           ↓                             │
│  ✏️ 脱敏编辑器（手动+智能建议）          │
│           ↓                             │
│  💾 保存到「脱敏相册」（明文存储）        │
│           ↓                             │
│  📤 分享/导出（系统分享面板）            │
│                                         │
└─────────────────────────────────────────┘
```

### 🔐 两个相册的区别

| 特性 | 原相册（Originals） | 脱敏相册（Redacted） |
|------|-------------------|---------------------|
| **存储内容** | 包含敏感信息的原图 | 脱敏处理后的图片 |
| **加密方式** | AES-256加密 | 不加密（明文） |
| **用途** | 安全存储、重新编辑 | 分享、导出 |
| **可见性** | 需FaceID/密码解锁 | 随时可见 |
| **编辑能力** | 可重新脱敏 | 只读（可删除） |

### 🔄 核心工作流

```
用户导入图片
    ↓
自动加密保存到「原相册」（后台静默完成）
    ↓
自动打开脱敏编辑器
    ↓
智能建议敏感区域（OCR识别，需用户确认）
    ↓
用户手动涂抹/调整脱敏区域
    ↓
保存到「脱敏相册」（明文PNG/JPEG）
    ↓
通过系统分享面板分享到微信/小红书等
```

## 6. 产品核心价值（Value Proposition）

### 1. **完全离线（Zero Network）**
- App内无任何网络功能
- 不请求网络权限
- 所有脱敏算法本地执行（Vision/CoreML）
- 不上传任何图片至服务器

### 2. **双重隐私保护（Dual Privacy Protection）**🆕
- **原图加密存储**：导入即加密，AES-256保护
- **脱敏图明文存储**：安全可分享，无需解密
- **密钥安全**：所有密钥存储在Keychain，系统级保护

### 3. **智能建议脱敏（Smart Suggestions）**🆕
- OCR识别文本区域（iOS Vision API）
- 智能标注可能的敏感信息：
  - 📱 手机号（11位数字）
  - 📧 邮箱地址
  - 🆔 身份证号（18位）
  - 💳 银行卡号（16-19位）
- **需用户确认后才脱敏**（避免误判）

### 4. **专业级图片脱敏工具**
- 手势涂抹马赛克（可调颗粒度）
- 矩形遮挡框（黑/白条，可调透明度）
- 高斯模糊（可调半径）
- 颜色遮挡（自定义颜色）

### 5. **工具优先的极简设计**🆕
- 启动即导入/编辑，不强调相册管理
- 相册功能作为"辅助入口"，不占主界面
- 专注脱敏工作流，避免功能蔓延

### 6. **零账号、零追踪（Zero Identity）**
- 不注册、不登录
- 不采集任何用户隐私
- 无广告、无埋点SDK

## 7. 产品核心原则（Product Principles）

✅ **必须遵守**：
1. 所有图片处理必须在本地执行（不得调用服务器）
2. 不添加任何需要网络的SDK（包括广告、数据埋点）
3. 原图必须加密存储（AES-256-GCM + Keychain）
4. 脱敏图片不加密（已安全，可自由分享）
5. 不支持视频（避免与ZeroNet Space功能重叠）
6. UI必须呈现"专业编辑器体验"，而非"私密空间体验"

⚠️ **允许但不强调**：
1. 本地文件库存在，但不作为主打功能
2. 相册管理功能，但UI上弱化展示
3. 批量处理功能，但保持简单（会话内批处理）

❌ **绝不添加**：
1. 访客模式（ZeroNet Space专属）
2. 伪装密码、双重空间（ZeroNet Space专属）
3. 视频脱敏（技术复杂度高，避免功能蔓延）
4. 云端同步、账号体系

## 9. 技术架构（Technical Architecture）

### 📦 核心技术栈

```
┌─────────────────────────────────────┐
│          应用层（SwiftUI）           │
├─────────────────────────────────────┤
│  导入模块  │  编辑模块  │  相册模块  │
├─────────────────────────────────────┤
│         业务逻辑层（Swift）          │
├─────────────────────────────────────┤
│ OCR识别 │ 图片编辑 │ 加密管理 │ 存储 │
├─────────────────────────────────────┤
│        iOS系统框架（Framework）      │
├─────────────────────────────────────┤
│ Vision │ CoreML │ CryptoKit │ Photos│
└─────────────────────────────────────┘
```

### 🔐 加密架构

```swift
// 原图加密流程
导入原图 → UIImage
    ↓
转换为Data → Data
    ↓
生成/获取密钥 → Keychain.getKey("master_key")
    ↓
AES-256-GCM加密 → EncryptedData
    ↓
存储到沙盒 → Documents/Originals/[UUID].enc
    ↓
存储元数据 → CoreData (id, thumbnail_enc, timestamp)
```

### 🗄️ 数据模型

```swift
// 原图模型
struct OriginalImage {
    let id: UUID
    let encryptedDataPath: String        // 加密原图路径
    let encryptedThumbnailPath: String   // 加密缩略图路径
    let createdAt: Date
    var redactedVersions: [UUID]         // 关联的脱敏图ID列表
}

// 脱敏图模型
struct RedactedImage {
    let id: UUID
    let imagePath: String                // 明文图片路径（PNG/JPEG）
    let originalImageId: UUID            // 关联的原图ID
    let redactedAt: Date
    let format: ImageFormat              // PNG / JPEG
}

// 智能建议模型
struct SensitiveRegion {
    let type: SensitiveType              // Phone, Email, IDCard, BankCard
    let boundingBox: CGRect              // 区域坐标
    let confidence: Float                // 识别置信度
    var isConfirmed: Bool = false        // 用户是否确认脱敏
}

enum SensitiveType {
    case phoneNumber    // 手机号
    case email          // 邮箱
    case idCard         // 身份证
    case bankCard       // 银行卡
    case custom         // 自定义
}
```

### 🧠 OCR识别流程

```swift
// 智能建议脱敏流程
加载图片 → UIImage
    ↓
Vision文字识别 → [VNRecognizedText]
    ↓
正则匹配敏感模式 → [SensitiveRegion]
    ↓
高亮显示建议区域 → UI绘制边框
    ↓
用户点击确认 → region.isConfirmed = true
    ↓
执行脱敏操作 → 应用马赛克/模糊
```

### 🎨 编辑器技术实现

- **马赛克**：Core Image `CIPixellate` 滤镜
- **模糊**：Core Image `CIGaussianBlur` 滤镜
- **遮挡**：Core Graphics 绘制矩形/路径
- **手势**：`UIPanGestureRecognizer` 绘制轨迹

## 11. 与ZeroNet Space的差异化

| 维度 | ZeroNet Space | ZeroNet Redact |
|------|--------------|---------------|
| **核心功能** | 隐私文件保险箱 | 文件脱敏工具 |
| **主要用途** | 存储原文件不修改 | 修改文件去除隐私 |
| **支持格式** | 照片、视频、文档 | 图片（V1.0）+ PDF（V1.5+）|
| **特色功能** | 访客模式、伪装密码 | 智能脱敏、双相册 |
| **UI定位** | 私密空间体验 | 专业编辑器体验 |
| **加密对象** | 所有存储的文件 | 仅原文件（脱敏文件明文）|
| **使用场景** | 长期保存私密文件 | 快速脱敏后分享 |

**结论**：两个产品互补，不冲突。ZeroNet Space是"保险箱"，ZeroNet Redact是"脱敏工具"。

## 12. 风险与挑战

### 技术风险
1. **OCR识别准确率**：可能误判或漏判敏感信息
   - 缓解：强调"智能建议"而非"全自动"，需用户确认
2. **加密性能**：大量图片加密可能影响性能
   - 缓解：异步加密，显示进度条
3. **缩略图加密**：影响相册列表加载速度
   - 缓解：考虑缩略图使用轻量级加密或模糊处理

### 产品风险
1. **功能蔓延**：容易变成"文件管理器"
   - 缓解：严格遵守"工具优先，相册次之"原则
2. **与ZeroNet Space功能重叠**
   - 缓解：明确差异化，不添加访客模式、视频支持等
3. **PDF支持复杂度**：PDF格式多样性（扫描版、加密、嵌入对象）🆕
   - 缓解：V1.5只支持纯文本PDF，V2.0逐步扩展

### 用户教育风险
1. **用户不理解双相册设计**
   - 缓解：首次使用引导，说明"原文件加密保护，脱敏文件可分享"
2. **PDF方案A vs 方案B差异**🆕
   - 缓解：V1.5明确说明"当前版本转图片，V2.0保持PDF格式"
