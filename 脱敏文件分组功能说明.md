# 脱敏文件分组功能说明

## 功能概述

为脱敏文件添加了完整的分组管理功能，支持自动继承分组、分组筛选和手动移动分组。

## 核心功能

### 1. 自动继承分组 ✅

**功能描述**：
- 导出脱敏文件时，自动继承原始文件的分组
- 无需用户手动设置，保持分组一致性

**实现位置**：
- `EditorViewModel.swift` 的 `exportFile()` 方法（约第329行）

**代码示例**：
```swift
if let originalFile = file as? OriginalFile {
    redactedFile.originalFile = originalFile
    // 继承分组
    redactedFile.group = originalFile.group
    print("✅ 脱敏文件继承分组: \(originalFile.group?.name ?? "无分组")")
}
```

### 2. 分组筛选查看 ✅

**功能描述**：
- 在AlbumView顶部显示分组选择器
- 点击分组可筛选显示该分组的脱敏文件
- 显示每个分组的文件数量
- 支持创建新分组和管理分组

**UI位置**：
- AlbumView顶部的横向滚动条
- 包含默认分组、自定义分组和"新建"按钮

**相关组件**：
- `RedactedGroupSelectorBar.swift` - 分组选择器组件
- `AlbumViewModel.swift` - 分组筛选逻辑

### 3. 手动移动分组 ✅

**功能描述**：
- 在脱敏文件预览页面，点击导航栏右侧的分组按钮
- 弹出分组选择器，显示所有可用分组
- 选择新分组后自动保存
- 当前分组有勾选标记

**UI位置**：
- RedactedFileDetailView导航栏右侧
- 显示当前分组图标和名称的胶囊按钮

**相关组件**：
- `RedactedFileGroupPicker` - 分组选择器弹窗
- `RedactedFileDetailView` 的 `moveToGroup()` 方法

## 数据模型变更

### Core Data 关系

**RedactedFile实体**：
- 新增 `group` 关系（可选，一对一）
- 指向 `FileGroup` 实体

**FileGroup实体**：
- 新增 `redactedFiles` 关系（可选，一对多）
- 指向 `RedactedFile` 实体的集合

### 删除规则
- 删除分组时：`Nullify`（脱敏文件的group字段置为nil）
- 删除脱敏文件时：`Nullify`（从分组中移除）

## 使用流程

### 完整工作流程

1. **导入文件到分组**
   - 在ImportView选择或创建分组
   - 导入图片或PDF文件

2. **编辑并导出**
   - 进行脱敏编辑（涂抹、马赛克等）
   - 点击"应用"导出脱敏文件
   - **自动继承原始文件的分组**

3. **查看脱敏文件**
   - 进入AlbumView（脱敏文件页面）
   - 顶部选择分组筛选
   - 点击文件查看预览

4. **移动分组**（可选）
   - 在预览页面点击右上角分组按钮
   - 选择新的分组
   - 自动保存并更新显示

## 技术实现细节

### 分组加载逻辑

```swift
private func loadGroups() {
    let fetchRequest: NSFetchRequest<FileGroup> = FileGroup.fetchRequest()
    fetchRequest.sortDescriptors = [NSSortDescriptor(keyPath: \FileGroup.createdAt, ascending: true)]
    
    do {
        allGroups = try viewContext.fetch(fetchRequest)
        currentGroup = redactedFile.group
    } catch {
        print("❌ 加载分组失败: \(error)")
    }
}
```

### 移动分组逻辑

```swift
private func moveToGroup(_ newGroup: FileGroup) {
    redactedFile.group = newGroup
    
    do {
        try viewContext.save()
        currentGroup = newGroup
        showGroupPicker = false
    } catch {
        print("❌ 移动分组失败: \(error)")
    }
}
```

### 分组筛选逻辑

```swift
func loadFiles() {
    var predicates: [NSPredicate] = []
    
    if let group = selectedGroup {
        predicates.append(NSPredicate(format: "group == %@", group))
    }
    
    if let type = filterType {
        predicates.append(NSPredicate(format: "fileTypeRaw == %@", type.rawValue))
    }
    
    if !predicates.isEmpty {
        request.predicate = NSCompoundPredicate(andPredicateWithSubpredicates: predicates)
    }
}
```

## 注意事项

⚠️ **重要提示**：
- 删除分组不会删除脱敏文件，只会将文件的group设为nil
- 移动分组是即时保存的，无法撤销（除非手动再次移动）
- 分组选择器显示的文件数量是实时计算的
